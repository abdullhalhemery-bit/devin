# Testing: Farcaster Mini App (devin)

## Overview
This app is a Farcaster Mini App (Frame v2) that allows users to claim $DEV tokens on Base and transfer FID on Optimism. It uses Farcaster-native wallet auth instead of external wallets.

## Environment
- **Production URL**: `https://devin-pi.vercel.app`
- **Preview URL**: Generated by Vercel for each PR branch
- **Framework**: Vite + React (single-page app)
- **Deployment**: Vercel with serverless API routes in `/api/`

## Devin Secrets Needed
- `SEED_PHRASE` — 12-word mnemonic used by API routes to generate addresses and sign transactions for FID transfers
- `NEYNAR_API_KEY` — API key for Neynar Farcaster API

These are set as Vercel environment variables. They are NOT needed for frontend testing but are required for API route testing.

## Key Testing Areas

### 1. Frame v2 / Mini App Embed Meta Tags
The most critical thing to verify when the embed config changes:

```bash
# Fetch the page and check for fc:miniapp meta tag
curl -s <PREVIEW_URL>/ | grep 'fc:miniapp'
```

**Must verify:**
- `fc:miniapp` meta tag exists with valid JSON
- JSON has `"type":"launch_frame"` (NOT `"link"` — this is the #1 cause of the app publishing as a plain link)
- JSON has `"title":"Claim $DEV"` (the button text shown in Farcaster feed)
- `fc:frame` tag also present (backward compatibility)
- NO old-style Frame v1 tags (`fc:frame:button:1:action`, `fc:frame` with `vNext`)

### 2. farcaster.json Manifest
**Important**: `vercel.json` redirects `/.well-known/farcaster.json` to Farcaster's hosted manifest API. The local file in `public/.well-known/farcaster.json` is a reference copy but NOT what gets served in production. To verify:

```bash
# This will follow the redirect to Farcaster's API
curl -sL <PREVIEW_URL>/.well-known/farcaster.json
```

If it returns `{"errors":[{"message":"Hosted miniapp manifest not found"}]}`, that means the manifest hasn't been registered with Farcaster's API yet — this is expected for preview URLs.

### 3. QR Code Wallet Auth (Browser Mode)
When opened in a regular browser (not inside Warpcast):
- App shows "Connect via Farcaster" button (no MetaMask/Rabby options)
- Clicking it opens a QR modal with:
  - Title: "Connect via Farcaster"
  - Subtitle about scanning with Warpcast
  - QR code SVG (green on dark background)
  - "Waiting for approval from Warpcast..." status
  - "Open in Warpcast" link with valid SIWF URL (`https://farcaster.xyz/~/siwf?channelToken=...`)
  - "Close" button
- Close button dismisses modal cleanly (no overlay remnants in DOM)

### 4. Mini App Auto-Connect (Warpcast Mode)
**Cannot be tested without physical Warpcast app.** When `window.frames?.sdk` is available (inside Warpcast), the app should auto-connect via `sdk.wallet.getEthereumProvider()` without showing any modal.

### 5. Transaction QR Modals
**Cannot be triggered without completing wallet auth first.** After connecting, Step 1 and Step 2 buttons should show QR codes in browser mode that redirect to Warpcast for transaction approval.

## Testing Limitations
- QR code scanning requires physical Warpcast app on a phone
- Mini-app auto-connect requires opening inside Warpcast
- Transaction flows require completing wallet auth first
- Farcaster embed preview (how it looks in the feed) can only be verified by sharing the URL in an actual Farcaster cast

## Common Issues
- If app publishes as a plain link on Farcaster: check that `fc:miniapp` meta tag has `"type":"launch_frame"` not `"link"`
- If QR code doesn't render: check that `@farcaster/auth-kit` and `qrcode.react` are installed
- If `farcaster.json` returns 404: check `vercel.json` redirect rule and that `public/.well-known/` directory exists
- If modal doesn't close: check that the `onClose` callback properly sets state to null
